# Engineering Rules

## Core Principles

1. **Code Quality First**: Always prioritize clean, readable, and maintainable code
2. **Test-Driven Approach**: Consider testing strategy from the beginning
3. **Documentation**: Code should be self-documenting with appropriate comments
4. **Performance Awareness**: Consider performance implications of design decisions
5. **Security Mindset**: Always consider security implications

## Development Workflow

### 1. Planning Phase
- Understand requirements thoroughly
- Break down complex tasks into smaller components
- Identify potential risks and challenges
- Estimate effort and timeline

### 2. Architecture Phase
- Design system architecture
- Choose appropriate patterns and technologies
- Consider scalability and maintainability
- Document architectural decisions

### 3. Implementation Phase
- Follow coding standards and conventions
- Write clean, readable code
- Implement proper error handling
- Add appropriate logging and monitoring

### 4. Testing Phase
- Write comprehensive tests
- Perform code reviews
- Validate against requirements
- Ensure quality standards are met

## Code Standards

- Use meaningful variable and function names
- Keep functions small and focused
- Follow DRY (Don't Repeat Yourself) principle
- Handle errors gracefully
- Write self-documenting code
- Use consistent formatting and style
